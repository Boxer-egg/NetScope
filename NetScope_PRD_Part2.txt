5.4 中栏 — 地图视图规格
地图配置
	•	地图类型：.standard（标准矢量地图），用户可右键切换 .hybrid（卫星+路网）
	•	关闭 showsUserLocation（用本机 IP 定位替代，不请求 CoreLocation 权限）
	•	关闭 isScrollEnabled 的惯性滚动（保持视角稳定），但允许用户手动拖拽平移
	•	关闭 showsCompass / showsScale（减少 UI 噪音）

本机原点 Annotation
	•	自定义 MKAnnotationView，直径 12pt 蓝色实心圆
	•	外圈 20pt 透明蓝色圆，以 0.8s 循环脉冲动画（scale 1.0 → 1.5 → 1.0，opacity 0.6 → 0）
	•	标注文字「YOU」，10pt，蓝色，圆点正下方 4pt

连线弧线渲染
	•	起点：本机原点坐标
	•	终点：目标 IP 的经纬度（GeoInfo.latitude / longitude）
	•	路径：二次 Bezier 曲线，控制点为起终点地理中点向北偏移 20°（模拟大圆弧）
	•	线宽：1.5pt（普通），2.5pt（选中进程的连线）
	•	非选中进程连线 opacity = 0.2，选中进程 opacity = 0.85
	•	新连线出现时：stroke-dashoffset 动画从全虚到实，时长 0.4s

目标 IP Dot Annotation
	•	直径 8pt 实心圆，颜色与连线同色
	•	点击弹出 Callout Bubble：显示 IP、城市、国家、ASN、端口
	•	同一坐标多个 IP 时，圆点稍微随机偏移（±0.5°）避免完全重叠

5.5 右栏 — 详情面板规格
顶部摘要区
	•	高度 64pt，背景略深于面板背景
	•	显示：进程名 + 图标（左）/ 连接数 · 国家数（右）
	•	选中「All」时显示全局统计

连接列表
	•	每行高 44pt
	•	左侧：6pt 色点（进程色）+ IP 地址（Monospaced 12pt）
	•	右侧：城市/国家（11pt 次要色）
	•	第二行：端口 · 协议 · 状态（10pt 次要色）
	•	点击行展开 Traceroute 面板（accordion 展开，动画 0.2s）

Traceroute 子面板
	•	每跳行高 36pt，左侧跳数序号（灰色，宽 24pt）
	•	纵向连线：每跳左侧 20pt 处，细虚线连接各跳圆点
	•	圆点颜色：绿（< 50ms）/ 黄（50-150ms）/ 红（> 150ms）/ 灰（timeout）
	•	超时跳显示「* * *  timeout」，RTT 列留空
	•	底部工具栏：「Re-run」按钮 + 完成状态文字 + 总跳数

6. Phase 2 — 进阶功能规格
范围
Phase 2 在 Phase 1 稳定后实施。所有功能均为独立模块，可按优先级顺序逐一开发，互不阻塞。

6.1 P2-F1：连接历史记录
功能描述
	•	记录每条连接的首次出现时间和最后活跃时间
	•	提供「历史模式」视图，可回溯过去 24 小时的连接记录
	•	支持按时间滑块回放，地图显示对应时间点的连接状态

数据存储
	•	SQLite 数据库存储于 ~/Library/Application Support/NetScope/history.db
	•	表结构：connections（id, pid, process_name, remote_ip, remote_port, proto, first_seen, last_seen, bytes_sent, bytes_recv）
	•	保留最近 7 天数据，超期自动删除
	•	写入操作在后台队列执行，不影响 UI 线程

6.2 P2-F2：流量统计
功能描述
	•	每个进程的出入流量统计（KB/s · MB 累计）
	•	左栏进程列表增加流量迷你图（最近 60 秒的 sparkline）
	•	详情面板顶部显示实时带宽

实现方案
	•	lsof 本身不提供流量字节数，需结合 netstat -p tcp 或读取 /proc（macOS 无 /proc）
	•	macOS 上使用 nettop -P -t wifi,en0 命令，按 PID 获取流量字节数
	•	或使用 Network Framework 的 NWPathMonitor（只有聚合数据）
	•	推荐：嵌入 libproc 的 proc_pidinfo 获取每进程流量（需 Private API 评估）
注意
macOS 上逐进程流量统计受沙盒限制，Phase 2 前需要做可行性验证，不能直接分发到 Mac App Store。

6.3 P2-F3：连接规则与告警
功能描述
	•	用户可为指定进程或 IP 段设置告警规则
	•	规则类型：「进程 X 连接到国家 Y 时弹出通知」/ 「任意进程连接到端口 443 之外时告警」
	•	触发后通过 macOS UserNotifications 发出系统通知
	•	规则管理界面（Preferences 窗口的独立 Tab）

6.4 P2-F4：多网卡 / VPN 感知
功能描述
	•	状态栏显示当前活跃网卡（en0 WiFi / en1 以太网 / utun0 VPN）
	•	VPN 激活时，地图上区分直连流量和 VPN 隧道流量（不同线型）
	•	检测到 VPN 时，显示 VPN 出口 IP 的地理位置，与本机实际 IP 区分标注

实现
	•	使用 NWPathMonitor 监听网络路径变化
	•	通过 getifaddrs() 枚举当前所有网络接口和 IP
	•	VPN 接口检测：utun 前缀且有 P2P flag 的接口

功能描述

6.6 P2-F6：连接聚合与去重显示
功能描述
	•	同一进程连接同一 IP 的多条记录，在地图和列表上合并为一条（显示连接数 N）
	•	同一 ASN（如「Amazon AWS」）的多个 IP，在地图上聚合为一个更大的 Annotation
	•	缩放地图时，Annotation 自动聚合/展开（类似 MapKit 的 Clustering）

实现
	•	使用 MKClusterAnnotation API，为所有 IP Annotation 设置相同 clusteringIdentifier
	•	自定义 cluster Annotation View，显示数量 Badge
